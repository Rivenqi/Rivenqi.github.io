Flutter Reverse

Flutter

Flutter是Google推出的，使用Dart语言开发的跨平台移动UI框架，可以在各平台上开发应用程序。

libapp.so格式

在发布版本的APK文件中，libapp.so保存的是快照，存放经过编译过的Dart业务代码，libflutter.so是引擎，其中包括一个dart的虚拟机。一个正向的过程是：flutter->虚拟机快照->ELF文件。

libapp.so逆向

由于libapp.so是使用Dart语言的AOT（静态提前编译）模式编译的库，只含有机器码，基本不含有函数信息，故直接将其拖入IDA中，函数名称及字符串含义无法直观看出，使得逆向难度增大。

1. 静态

Doldrums

目前市面上静态解析工具有2个，一个是Doldrums(https://github.com/rscloura/Doldrums)，这个项目比较老而且不再更新了，目前只支持两个dart版本，而且这两个版本比较老。

blutter

目前这个工具只支持64位的flutter

将目标apk中的libflutter.so和libapp.so抽取出来，置于一个文件夹下

在x64 Native Tools Command Prompt for VS 2022（在Visual Studio2022中带有）运行

output文件夹下：

  1. 用IDA打开libapp.so
  2. 先将iad_dart_struch.h导入IDA中，File->Load file->Parse C header file
  3. 在IDA中执行addNames.py

随后可在IDA中还原出函数名，方便逆向分析：

对于常规的AES加密，可直接在pp.txt中找到解密密钥与IV

reflutter

动态解析工具最具代表性的是reflutter（https://github.com/Impact-I/reFlutter）。

流量拦截

​ 因为Dart使用Mozilla的NSS库生成并编译自己的Keystore，导致我们不能通过将代理CA添加到系统CA存储的方式来绕过SSL验证。reFlutter框架使用了已编译且重新封装的Flutter库来帮助广大研究人员对Flutter APP进行逆向工程分析，通过socket.cc可以执行流量拦截和监控。relflutter使用时需要输入burpsuite的监听IP和端口，得到release.RE.apk，此apk尚未签名，需手动签名。

​ 下一步，按照reflutter的提示，设置burpsuite的代理，绑定所有地址，勾选支持不可见代理。设置Drony的wifi代理主机名端口和burpsuite一致。随后就能抓到HTTPS请求了。

​ 由于使用reflutter需要重新打包app，所以其局限性比较大。

dump code

​ 同上述操作，选择选项2，将release.RE.apk签名后，安装至手机

即可还原出dart代码，但也同样由于需要重新打包app，大部分app有防重打包机制，重打包后闪退导致无法还原出dart代码，需绕过防重打包。
