IDA动态调试Android So

IDA动态调试So

环境准备

首先在ida的安装目录下找到对应的 android_server 文件,如果手机是64位的就使用 android_server64 ，拷贝到手机 /data/local/tmp 目录下面。

adb push android_server /data/local/tmp

修改文件属性可执行

转发端口

adb forward tcp:23946 tcp:23946

Tip:

有些软件会检测这个端口，所以可以把android_server文件改名，然后在随意某个自定端口启动

./android_server -p23333

启动DDMS

monitor.bat

调试方式启动APK

启动后，手机端出现Waiting For Debugger弹窗

IDA连接

Debugger -> Attach -> Remote Arm Liinux/Android Debugger

Debugger setup

勾选

jdb挂载

在DDMS上选择相应进程后，使用指令使apk继续运行，成功后，DDMS上进程将显示绿色，否则是红色

jdb -connect com.sun.jdi.SocketAttach:hostname=127.0.0.1,port=8700

快捷键

故障排除

IDA报错

0: got SIGSEGV signal (Segmentation violation) (exc.code b, tid 23425)

一般为程序未打开调试或程序有反调试措施

DDMS无法正常读取app列表

一般由于ro.debuggable值为0导致

解决方法

  • 检查设备ROOT是否正常
  • 将对应版本的mproppush至/data/local/tmp下
  • chmod 777 mprop添加执行权限
  • 执行./mprop ro.debuggable 1

再次查看ro.debuggable

重启生效stop;start
